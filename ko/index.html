이메일로 OpenCode
원격 제어

MailCode는 당신의 받은편지함과 OpenCode(오픈소스 AI 코딩 에이전트)를 연결합니다. 작업 전송, 결과 수신, 명령 주입을 모두 이메일로 처리하세요. 서드파티 의존성 제로, 한 줄 명령으로 설치.

알림

작업 완료 시 이메일 알림. 답장으로 다음 작업을 계속합니다.

두 가지 사용 모드

OpenCode 실행 여부에 관계없이 MailCode가 지원합니다.

답장 모드

OpenCode 실행 중

OpenCode가 작업을 완료하면 브리지 플러그인이 유휴 상태를 감지하고 8자 토큰이 포함된 이메일 알림을 발송합니다. 해당 이메일에 답장만 하면 다음 명령을 보낼 수 있습니다. 토큰이 올바른 세션으로 라우팅합니다.

당신의 답장: #A3B7KM9Q parse_config 함수 최적화 계속
콜드 스타트

OpenCode 미실행

session_key와 작업 설명이 포함된 새 이메일을 보냅니다. MailCode가 자동으로 tmux 세션을 생성하고 대상 프로젝트에서 OpenCode를 실행한 후 명령을 주입합니다. 확인 메일 수신 후 답장 모드로 전환됩니다.

새 메일: key: a1b2c3d4e5f67890 project: /path/to/your-project helper.py의 parse_config 함수 리팩토링

MailCode의 차별점

어디서든 코드를 제어해야 하는 개발자를 위해 설계되었습니다.

📡

실시간 IMAP IDLE

폴링 지연 없음. IMAP IDLE 장기 연결로 초 단위 메일 감지.

🔐

DKIM + SPF 인증

DNS 수준 이메일 인증으로 발신자 확인. 엄격 모드에서 미검증 발신자 차단.

📢

다중 채널 알림

이메일, macOS 알림, Slack, Discord, Feishu, DingTalk, WeCom, 커스텀 Webhook.

⏱

예약 작업

Cron 같은 이메일 스케줄링으로 정기 자동화 구현.

💻

CLI + TUI 이중 인터페이스

Textual 기반 터미널 UI와 강력한 CLI를 동시에 제공.

📦

서드파티 의존성 제로

순수 Python 표준 라이브러리만 사용. pip install 불필요. Python 3.9+와 tmux만 있으면 됩니다.

보안 최우선

다중 보호 계층으로 당신만이 OpenCode 세션을 제어할 수 있습니다.

🚫

명령 블랙리스트

rm -rf, sudo rm, 파이프 실행 등 위험한 명령을 기본 차단.

✅

발신자 화이트리스트

허용된 이메일 주소만 명령 전송 가능. 나머지는 무시.

🔑

Session Key 인증

콜드 스타트에는 128비트 랜덤 session_key가 필요합니다.

⏳

세션 만료 및 제한

세션은 24시간 후 자동 만료. 명령 실행 횟수 제한 있음.

한 줄로 설치

필요 사양: Python 3.9+, tmux, OpenCode CLI. 플랫폼을 선택하세요.

brew install python3 tmux
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

설치 후 ~/.config/mailcode/config.json을 편집하고 mailcode serve --idle을 실행하세요.

apt update && apt install python3 tmux -y
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

설치 후 ~/.config/mailcode/config.json을 편집하고 mailcode serve --idle을 실행하세요.

dnf install python3 tmux -y
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

설치 후 ~/.config/mailcode/config.json을 편집하고 mailcode serve --idle을 실행하세요.

pacman -S python tmux --noconfirm
