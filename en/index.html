How It Works

MailCode creates a simple bridge between your email inbox and OpenCode running in tmux.

✉️

You Send an Email

A coding task or command, sent from any email client — phone, tablet, or laptop.

→
📡

IMAP Listener

MailCode polls your inbox in real time via IMAP IDLE, detecting new messages instantly.

→
⚡

tmux Injection

Commands are injected into your running OpenCode tmux session via send-keys.

→
📬

Notification

When the task completes, you receive an email summary — reply to continue the loop.

Two Ways to Use It

Whether OpenCode is already running or not, MailCode has you covered.

Reply Mode

When OpenCode Is Already Running

OpenCode completes a task in its tmux session. The bridge plugin detects the idle state and sends you an email notification with a unique 8-character token. Simply reply to that email with your next instruction — the token routes your command to the right session, and MailCode injects it via tmux send-keys.

Your email reply: #A3B7KM9Q Continue optimizing the parse_config function
Cold Start Mode

When No OpenCode Is Running

Send a new email with your session key and task description. MailCode automatically creates a new tmux session, launches OpenCode in the target project directory, and injects your command. You receive a confirmation email with the new token — subsequent replies use Reply Mode.

Your email (new message): key: a1b2c3d4e5f67890 project: /path/to/your-project Refactor helper.py's parse_config function

What Makes MailCode Different

Built for developers who need to stay in control, from anywhere.

📡

Real-Time IMAP IDLE

No polling delays. Uses IMAP IDLE long connection for instant email detection, so commands arrive within seconds.

🔐

DKIM + SPF Auth

Validates sender authenticity through DNS-level email authentication. Optional strict mode rejects unverified senders.

📢

Multi-Channel Notifications

Email, macOS desktop notifications, Slack, Discord, Feishu, DingTalk, WeCom, and custom webhooks — choose your channel.

⏱

Scheduled Tasks

Set up cron-like email scheduling for recurring automation — nightly backups, daily reports, scheduled code reviews.

💻

CLI & TUI Dual Interface

Full-text terminal UI (Textual) for live monitoring plus a powerful CLI for scripting and automation integration.

📦

Zero Third-Party Dependencies

Pure Python standard library — imaplib, smtplib, subprocess, json. No pip install required. Just Python 3.9+ and tmux.

Security First

Multiple layers of protection ensure only you control your OpenCode sessions.

🚫

Command Blacklist

Dangerous commands like rm -rf, sudo rm, and pipe-to-shell patterns are blocked by default.

✅

Sender Whitelist

Only email addresses you explicitly allow can send commands. Everything else is silently ignored.

🔑

Session Key Auth

Cold start requires a 128-bit random session key. Without it, no OpenCode session can be launched remotely.

⏳

Session Expiry & Limits

Sessions auto-expire after 24 hours. Each session has a command execution limit. Old sessions are cleaned automatically.

Install in One Command

Requires Python 3.9+, tmux, and OpenCode CLI. Choose your platform.

brew install python3 tmux
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

After installation, edit ~/.config/mailcode/config.json with your email credentials, then run mailcode serve --idle.

apt update && apt install python3 tmux -y

Real Scenarios

Developers use MailCode to stay connected to their codebase no matter where they are.

🔍

Remote Code Review

CI pipeline fails while you are away from your desk. MailCode notifies you via email, you reply with a fix command, and OpenCode handles it — all from your phone.

Scheduled Automation

Production issue while commuting? Send an email with the hotfix command. MailCode cold-starts OpenCode on your home server, applies the patch, and confirms the result.
