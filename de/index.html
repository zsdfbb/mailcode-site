MailCode verbindet Ihr Postfach mit OpenCode — dem Open-Source-KI-Coding-Agenten. Senden Sie Aufgaben, empfangen Sie Ergebnisse und injizieren Sie Befehle über jedes E-Mail-Konto. Null Abhängigkeiten von Drittanbietern. Ein Befehl zur Installation.

Zwei Verwendungsmodi

Ob OpenCode bereits läuft oder nicht, MailCode hat die passende Lösung.

Antwort-Modus

Wenn OpenCode bereits läuft

OpenCode schließt eine Aufgabe in tmux ab. Das Bridge-Plugin erkennt den Leerlauf und sendet Ihnen eine E-Mail-Benachrichtigung mit einem eindeutigen 8-stelligen Token. Antworten Sie einfach auf diese E-Mail mit Ihrem nächsten Befehl.

Ihre Antwort: #A3B7KM9Q Optimierung der parse_config-Funktion fortsetzen
Kaltstart-Modus

Wenn OpenCode nicht läuft

Senden Sie eine neue E-Mail mit Ihrem Session-Key und der Aufgabenbeschreibung. MailCode erstellt automatisch eine tmux-Sitzung, startet OpenCode im Zielprojektverzeichnis und injiziert Ihren Befehl.

Ihre E-Mail (neue Nachricht): key: a1b2c3d4e5f67890 project: /path/to/your-project Refaktorieren Sie parse_config in helper.py

Was MailCode auszeichnet

Entwickelt für Entwickler, die von überall die Kontrolle behalten müssen.

📡

Echtzeit IMAP IDLE

Keine Polling-Verzögerung. IMAP IDLE Dauerverbindung für sofortige E-Mail-Erkennung.

🔐

DKIM + SPF Authentifizierung

Überprüft die Authentizität des Absenders über DNS. Optionaler Strikt-Modus weist unbestätigte Absender zurück.

📢

Multi-Kanal-Benachrichtigungen

E-Mail, macOS-Desktop, Slack, Discord, Feishu, DingTalk, WeCom, benutzerdefinierte Webhooks.

⏱

Geplante Aufgaben

Cron-ähnliche E-Mail-Planung für wiederkehrende Automatisierungen — nächtliche Backups, tägliche Berichte.

💻

CLI & TUI Doppelschnittstelle

Textual-basiertes Terminal-UI für Live-Überwachung plus leistungsstarkes CLI für Skripting.

📦

Keine Drittanbieter-Abhängigkeiten

Reine Python-Standardbibliothek — imaplib, smtplib, subprocess. Kein pip install erforderlich.

Sicherheit an erster Stelle

Mehrere Schutzschichten stellen sicher, dass nur Sie Ihre OpenCode-Sitzungen kontrollieren.

🚫

Befehls-Blacklist

Gefährliche Befehle wie rm -rf, sudo rm werden standardmäßig blockiert.

✅

Absender-Whitelist

Nur explizit erlaubte E-Mail-Adressen können Befehle senden. Alles andere wird ignoriert.

🔑

Session-Key-Authentifizierung

Kaltstart erfordert einen 128-Bit-Zufalls-Session-Key. Ohne diesen kann keine OpenCode-Sitzung remote gestartet werden.

⏳

Sitzungsablauf und -grenzen

Sitzungen laufen nach 24 Stunden automatisch ab. Jede Sitzung hat ein Befehlslimit. Alte Sitzungen werden automatisch bereinigt.

Derzeit Kostenlos

Alle Funktionen inklusive. Keine versteckten Limits. Keine Kreditkarte erforderlich.

Mit einem Befehl installieren

Erfordert Python 3.9+, tmux und OpenCode CLI. Wählen Sie Ihre Plattform.

brew install python3 tmux
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

Bearbeiten Sie nach der Installation ~/.config/mailcode/config.json und führen Sie mailcode serve --idle aus.

apt update && apt install python3 tmux -y
git clone https://github.com/zsdfbb/mailcode.git
cd mailcode
bash install.sh

Bearbeiten Sie nach der Installation ~/.config/mailcode/config.json und führen Sie mailcode serve --idle aus.

dnf install python3 tmux -y
git clone https://github.com/zsdfbb/mailcode.git

Anwendungsszenarien

Entwickler nutzen MailCode, um von überall mit ihrer Codebasis verbunden zu bleiben.

🔍

Remote-Code-Review

CI-Pipeline schlägt fehl, während Sie nicht am Schreibtisch sind. MailCode benachrichtigt Sie per E-Mail, Sie antworten mit einem Fix-Befehl, OpenCode führt ihn aus — alles vom Telefon.

🔄

Geplante Automatisierung

Konfigurieren Sie cron-ähnliche E-Mail-Trigger für nächtliche Datenbank-Backups, Abhängigkeitsupdates oder automatisiertes Refactoring.

🚨

Notfall-Hotfixes unterwegs

Produktionsproblem während des Pendelns? Senden Sie eine E-Mail mit dem Hotfix-Befehl. MailCode startet OpenCode auf Ihrem Heimserver, wendet den Patch an und bestätigt das Ergebnis.
